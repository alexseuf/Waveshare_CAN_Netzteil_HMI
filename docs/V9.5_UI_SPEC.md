# V9.5 – UI-Spezifikation Einstellungen

Status: freigegebenes Zielbild für die Umsetzung auf dem Branch `feature/v9.5`.

## Hauptseite Einstellungen

Die bisherige Einstellungsseite wird durch drei große Kacheln ersetzt:

1. **Debug Settings**
2. **WiFi**
3. **MQTT / Home Assistant**

Unterhalb der Kacheln befindet sich eine kompakte Diagnosezeile mit:

- freiem RAM
- freiem PSRAM
- CPU-Last
- LVGL-FPS
- Systemlaufzeit

Die untere Hauptnavigation bleibt sichtbar. Die aktive Seite `Einstellungen` wird blau hervorgehoben.

## Debug Settings

Linke Spalte:

- Log-Level
- serielles Debug
- Debug-Log auf Display
- FPS-Anzeige
- RAM-/CPU-Anzeige
- CAN-Rohdaten protokollieren
- optionales SD-Karten-Logging
- Schaltfläche `Log löschen`

Rechte Spalte Systemdiagnose:

- freier RAM
- freier PSRAM
- CPU-Last je Kern oder zusammengefasst
- LVGL-FPS
- Loop-Zeit
- Uptime
- Reset-Grund
- Schaltfläche `Diagnose aktualisieren`

## WiFi

Linke Spalte Status:

- WLAN aktiviert/deaktiviert
- SSID
- Signalstärke in dBm und Prozent
- IP-Adresse
- Gateway
- NTP-Status
- letzte Synchronisation
- Internetstatus
- Schaltfläche `WiFi trennen`

Rechte Spalte Konfiguration:

- Netzwerkauswahl
- erneute Netzwerksuche
- Passwortfeld mit Sichtbarkeitsschalter
- DHCP beziehungsweise später statische IP
- Schaltfläche `Speichern & verbinden`

Untere Aktionen:

- `Netzwerke suchen`
- `Access Point starten`

## MQTT / Home Assistant

Linke Spalte MQTT-Broker:

- aktiviert/deaktiviert
- Broker-Adresse
- Port
- Benutzername
- Client-ID
- Topic-Präfix
- Keepalive
- Status
- letzte Verbindung
- letzter Fehler
- `Verbindung testen`
- `Stopp` beziehungsweise `Start`

Rechte Spalte Home Assistant:

- Discovery aktivieren
- Geräte-ID
- Gerätename
- Availability/LWT
- Steuerbefehle erlauben
- Discovery erneut senden
- Speichern

`Steuerbefehle erlauben` ist nach Werkseinstellung deaktiviert.

## Zugriffsschutz

- Statuswerte und Diagramme dürfen optional ohne Anmeldung lesbar sein.
- START, STOP, Sollwertänderungen und Konfiguration erfordern das Administratorpasswort.
- Webzugriff wird auf das lokale Netzwerk beschränkt.
- Lokale Bedienung am Touchdisplay hat Vorrang vor Web- und MQTT-Befehlen.

## Fehlertoleranz

- WLAN-, NTP-, MQTT- und Webserverfehler dürfen CAN, Ladebetrieb, Display oder Diagrammpuffer nicht blockieren.
- Keine blockierenden Verbindungswarteschleifen im UI- oder CAN-Pfad.
- Netzwerkdienste arbeiten ereignisgesteuert und mit begrenzten Warteschlangen.

## Visuelle Vorgaben

- dunkler Hintergrund entsprechend der bestehenden HMI-Oberfläche
- abgerundete Panels mit feiner graublauer Kontur
- Blau als allgemeine Aktionsfarbe
- Grün für aktive und verbundene Zustände
- Rot für Trennen, Stopp und Löschen
- Violett für MQTT/Home Assistant
- große Touchflächen für die 800×480-Auflösung
- Statuswerte werden nicht nur über Farbe, sondern zusätzlich als Text dargestellt
